## What This Is

A Next.js 16.2.6 + Supabase + Stripe portal for auto transport booking and carrier dispatch. This is the Full Stack layer that sits on top of an n8n automation engine. Both layers share the same `order_guid` contract, same HubSpot custom properties, and same Supabase schema — built by one team with no integration gap.

## Critical Architecture Decisions

**order_guid** — Every shipment gets a UUID generated by the portal on creation. This UUID is sent with every n8n webhook call, written to HubSpot as a custom property, and used to sync status updates back to Supabase. It is the correlation key between all three systems. Never remove it.

**Two booking models** — Full Service (platform assigns carrier) and Carrier Marketplace (carriers bid). Separate n8n pipelines handle each. The portal captures the user's choice and passes it in the webhook payload.

**Three service tiers** — Standard, Priority, White Glove. White Glove triggers dedicated automation nodes for daily SMS and senior coordinator assignment.

**Realtime** — Uses Postgres Changes mode, not Broadcast. Broadcast caused the Supabase outage June 24-25 2026. 30-second fallback polling is baked into `hooks/use-realtime-status.ts` — never remove it.

**Vehicle types** — All 19 Super Dispatch enum values are defined in `lib/types/vehicle.ts`. The DB schema enforces them with a CHECK constraint. Sending `car` returns HTTP 400 from Super Dispatch — breaking change verified March 20 2025.
